The expected "data" objects included in the SessionResult objects resulting from the unit test, partitioned by session and Test.

fetchConfigResults

fetchConfigResults: PromiseResult[]

Represents the result of zero or more calls to Provider.fetchSessionConfig.

remarks

These are not the actual results of the call, but simply an array of PromiseResult specifying whether the Promise resolves or rejects. The unit being tested doesn't really use the result, so we don't probe that here.
