createFetchTestResult

  • createFetchTestResult(timingEntry: ResourceTimingEntry, response: Response, testConfig: TestConfiguration, setupResult: TestSetupResult): Promise<TestResultBundle>
  • remarks

    A provider implements this in order to define its logic for creating a {@link ResultBundle} describing the outcome of running a Fetch test.

    Parameters

    • timingEntry: ResourceTimingEntry

      The Resource Timing entry used to generate the test result.

    • response: Response

      The Response object resulting from the fetch activity.

    • testConfig: TestConfiguration

      The test configuration.

    • setupResult: TestSetupResult

      The provider-defined result of any test setup activity done prior to the fetch.

    Returns Promise<TestResultBundle>

getResourceUrl

  • getResourceUrl(testConfig: TestConfiguration): URL

  • A hook enabling a provider to generate a test URL at runtime.

    remarks

    A provider may cache the result of the first call to this method and reuse the value on subsequent calls.

    Parameters

    • testConfig: TestConfiguration

      The test configuration, which usually specifies a base URL from which the provider produces the runtime URL.

shouldRun

  • shouldRun(): Promise<boolean>

  • A hook called very early by the core module, enabling providers an opportunity to determine if they should participate in the session, e.g. based on user agent feature detection, random downsampling, etc. It returns a promise, in case a provider would like to make a network request in order to make this determination.

    Returns Promise<boolean>
